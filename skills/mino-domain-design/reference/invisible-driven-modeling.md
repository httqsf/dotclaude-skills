# invisible-driven-modeling — 見えない概念のモデリング

> 由来：本人の登壇スライド「見えないものに着目すると上手くいく、モデリングの勘所 / invisible-driven-design」に由来する固有概念。

## 核心
人間は入力情報の約9割を視覚に頼るため、思考が **「物理的に見えるもの（犬・猫・商品・ユーザー）」に引っ張られ**、それをそのままクラス化して巨大・複雑なモデルを生む（アンコンシャスバイアス）。これに抗い、**目的・問題・出来事・関係性といった「目に見えない概念」を顕在化させて** モデル化することが変更容易性を高める鍵。合言葉は「目を閉じ、心眼を開け」。

## 実践指針（本人の8ノウハウ要約）
- 目的ベースで抽象化する（対象の特徴のうち、その目的に合う特徴だけを抜き出す）。
- 物理的存在ではなく目的特化の概念で解釈する（「商品の売買」→所有権の移転、「貨物輸送」→船荷証券）。
- ユースケース分析・イベントストーミングで、見えない目的や問題を炙り出す。
- 些細な目的差を見破り、フラグではなく個別モデルへ分離する。
- **異常系ファースト**で検討する（異常系は問題解決ルール＝条件分岐の宝庫）。
- **出来事（注文→請求→キャンセル）** を個別モデル化し、時系列の矛盾を防ぐ（イミュータブルデータモデリング）。
- **関係性** を交差エンティティでモデル化し、Null許容カラムを排除する。

## 具体例
「犬・猫」という物理存在をそのままクラスにするのではなく、目的が「ペット販売」なら `ペット（販売品）` モデル、「血統管理」なら `血統書` モデルとして、見えない目的を軸にモデルを立てる。巨大 `User` クラス（身体特性〜心理状態まで混在）も目的特化モデルへ分解する。
