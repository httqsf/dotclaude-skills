# t-wada-quality-mindset — 品質・工学哲学

t-wada（和田卓人）の、品質と速度・自動テスト・リファクタリング・レガシー・堅牢性を貫く工学哲学。技術判断の「なぜ」を支える上位レイヤ。

> 貫く一言: **内部品質への投資は速度として回収される**。自動テストの目的はコスト削減でなく「変化し続ける力」。

## いつ・どう使うか
- 「品質より速度優先で」と言われた → `quality-and-speed`（内部品質と速度は対立しない）
- 自動テストの導入目的を問われた → `automated-testing-value`
- リファクタが怖い/止められている → `fearless-refactoring`
- テストのない巨大な既存コードに着手する → `legacy-code-rescue`
- 例外・表明・契約の方針を決める → `robust-code-by-contract`

## クイック原則
- 「保守性を高めるのは数年後の誰かでなく、**1ヶ月後の自分**のため」
- 損益分岐点：内部品質投資は**1ヶ月以内**、テスト自動化は**約4回**の実行で回収
- 「**テストのないコードはレガシーコード**」（M.フェザーズ）
- **テスト自体は品質を上げない**——品質を上げる"きっかけ"。上げるのはプログラミング＝リファクタリング
- 堅牢性は「**予防 > 対処**」。ミクロは正当性(fail fast)、マクロは堅牢性

> 注: 「質とスピード」の"品質"は**内部品質**（保守性）。外部品質（機能・正しさ）を落とせという意味ではない。
