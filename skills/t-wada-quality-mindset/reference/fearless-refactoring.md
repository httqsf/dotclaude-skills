# fearless-refactoring — 恐れないリファクタリング

## 核心
リファクタリングとは「外部から見た振る舞いを変えずに内部構造を改善すること」。自動テストという安全網（セーフティネット）があるからこそ、リグレッションを恐れずコードを変更できる。**テスト自体が品質を上げるのではなく、テストは品質を上げる"きっかけ"**であり、実際の品質はプログラミング＝リファクタリングによって上がる。

## 指針
- リファクタ前に「**実装から間合いを取ったテスト**」（実装詳細に密結合しない）を用意し、外部振る舞いを固定する（→ `t-wada-test-reliability/reference/fragile-tests.md`）。
- テストは天秤（スケール）と同じ——問題を可視化するが、問題自体は解決しない。設計変更で解決する。
- リファクタリングに強いテスト（実装変更で壊れないテスト）を書くこと自体が設計改善につながる。
- 「テストでは品質は上がらない。テストはあくまでも品質を上げるきっかけ」。
- 安全網が整えば、変更に踏み切れる＝レガシーコード改修で最も怖いリグレッションのリスクを下げられる。
