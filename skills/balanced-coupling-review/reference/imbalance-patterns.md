# imbalance-patterns — 不均衡パターン判定表（正典）

## 核心

不均衡は「強度と距離が同値（両方高い or 両方低い）」のときに生まれ、変動性がそれを実害化するか無害化するかを決める。単なる密結合批判ではなく、**低凝集（弱×近）も対称的に検出する**のが本モデルの独自性。「疎結合と高凝集は別々の概念ではなく、どちらも均衡結合の一形態」（BALANCE式の含意。詳細は `balanced-coupling-core/reference/balance-formula.md`）。

## 強度×距離の4象限

| | 低距離 | 高距離 |
|---|---|---|
| **低強度** | **低凝集（複雑性）** — 無関係なコンポーネントの同居。認知負荷増。big ball of mud へ漂流 | **疎結合（モジュール性）** — 低強度がカスケード変更を最小化。均衡 |
| **高強度** | **高凝集（モジュール性）** — 共進化必須だが低距離ゆえカスケードが安い。均衡 | **密結合（複雑性）** — 頻繁なカスケード（高強度）×高い実装コスト（高距離）。分散モノリスへの一歩 |

モジュール性は強度と距離が互い違い（XOR成立）のとき、複雑性は同値のときに生まれる。

## 変動性を加えた判定表（Severity直結・本ファミリーの正典）

```
強い＋近い＋高変動 → 高凝集。共同変更しやすく合理的（均衡。指摘しない）
弱い＋遠い＋高変動 → 疎結合。契約と互換性が管理されていれば成立（均衡。指摘しない）
強い＋遠い＋高変動 → Critical。分散モノリス。変更は頻繁・高コスト・予測不能。最優先で直す
強い＋遠い＋低変動 → 許容可能な技術的負債。記録するが優先しない（Minor）
弱い＋近い       → 低凝集の疑い。変動性に関わらず認知負荷の問題（Minor〜Significant）
```

- **Critical** = 高強度×高距離×高変動のみ。複雑性の能動的な発生源
- **Significant** = 中程度に変動的な領域の不均衡、または統合点を隠す暗黙的結合。進化とともに痛む
- **Minor** = 低変動領域の不均衡、またはカスケードは起こさないが認知負荷を上げる低凝集。機会対応

フラグ対象は「**不均衡かつ変動的**」のみ。XOR不成立＝即問題ではなく、NOT VOLATILITY で救済されるかを必ず確認してから深刻度を付ける。

## 3大リスクの検出兆候

**知識漏洩（Knowledge Leakage）** — 統合強度の異常として検出:
- 相手の private インターフェース利用（内部DB直接参照・privateオブジェクト・非公開API）。侵入された側は統合の存在すら知らない可能性があり、最も暗黙的で脆弱
- ビジネスルールの重複実装（例: 同じバリデーションがフロントとバックに二重実装され、仕様変更が片方だけ反映されて不整合化）＝機能結合の極端形
- ドメインモデル全体の共有（配送コンポーネントが請求・与信フィールドまで受け取る等）
- 常に「共有知識は暗黙的か明示的か」を問う。暗黙的結合が特に危険

**カスケード変更（Cascading Changes）** — 高強度の帰結:
- 共有する知識が多いほど、その一部が変わる確率が上がり、変更を相手へ伝達せざるを得なくなる
- 兆候: 1つの仕様変更で複数コンポーネント/リポジトリの修正が常態化。変更の影響先が事前に列挙できない（変更してみないと結果が分からない状態）

**分散モノリス（Distributed Monolith）** — 高強度×高距離の帰結:
- 多くのマイクロサービス移行は、共有知識を減らさずに距離だけを増やし、ローカルな複雑性をグローバルな複雑性に交換しただけ（Khononov, InfoQ）
- 兆候: サービスは分かれているのに同時デプロイが必要／リリース順序の調整が要る／同期API連鎖で障害が波及／API変更のたびに利用者側も全部直す

## 実践指針

- 全部を指摘しない。「不均衡×変動的」の交差だけを Critical/Significant に上げ、低変動の不均衡は Minor として記録に留める
- 「とにかく分割」を提案しない。分割は距離を増やす操作であり、強度が十分低いかライフサイクル結合が主問題のときだけ有効
- 密結合だけでなく低凝集（弱×近）も対称的に検出する
- 非同期化は実効距離を増やす。強度低減（契約化）とセットでないとバランスは悪化する
- 組織次元を必ず織り込む: 同じコード構造＋別チーム＝実効距離が高い（Conway's Law）
- TypeScriptでもGoでもJavaでも判定表は同じ。言語ではなく「共有知識×配置×変わりやすさ」を見る
