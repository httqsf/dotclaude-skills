# tddbc-shakyo-mobbing — TDDBC・写経・モブと若手訓練

## 核心
学習は「**真似から入る**」ことを勧める。写経（コードを書き写しながら読み進める）やチュートリアルで身体に入れるのが基本。TDDはプログラミングの手順を明確に定義した数少ない手法で、小さなステップをくるくる回すため、TDDを経験することで「**問題を分解して一つひとつ倒していく**」という若手に不足しがちな仕事術そのものが身につく。

## 指針
- まず真似る：写経・チュートリアルでライブコーディングを追体験する。
- TDD Boot Camp（2009年立ち上げ）のように、座学だけでなく実習形式で手を動かして体得する。
- ペアプロ・モブプロで、経験者の思考プロセス（TODOへの分解、Red→Green→Refactor）を共有する。
- 「**乱取り**」形式（相手・言語を替えながら短時間で交代して数をこなす）で経験の絶対量を稼ぐ。
- 若手のタスク管理の悩み（分解の順序が掴めない）に対し、TDDのTODOリスト分解を仕事術として転用させる。
- 最初のTODOリストは「だいたい間違っている」前提で、フィードバックで方向修正していく姿勢を教える。

## 取り組み例
- t-wada本人の原体験：夜行バスの中でKent Beck『Test-Driven Development』の生原稿を写経しながら読み進めた。
- TDD Boot Camp を全国で開催（基調講演＋ライブコーディング＋ペアプロ実演＋各言語のdojo＋乱取り実習）。
- 複数企業の技術顧問として、新卒・インターンとペアプログラミングを実践。
