# clean-code-that-works — 動作するきれいなコード

## 核心
TDDのゴールは Ron Jeffries の「動作するきれいなコード（clean code that works）」。これは「動作する（works）」と「きれい（clean）」の複合難問で、TDDの本質は**この難問を2つに分割して片方ずつ倒す分割統治（divide and conquer）**にある。

## 指針
- ゴールへの道は2つ。**オレンジのルート**＝「きれいな設計→実装」（理屈は正しいが**分析麻痺**に陥りやすい）。**青いルート**＝「動作するコード→きれいにする」（TDDが採る）。
- 青いルートの最大の障壁は技術でなく**恐怖（fear）**。特に「動くコードに触れるな」という業界的マインドセット。
- 恐怖には蛮勇でなく道具で対抗する。支える3本柱＝**バージョン管理・テスティング・自動化**。
- 「きれい」と「動作する」を同時に狙わない。まず動かし、テストという安全網の下できれいにする。
- 完璧に最初から書ける人はいない、という現実を出発点にする。
