# mino-organization — ミノ駆動流 設計文化の組織展開

個人の設計スキルを、チーム・組織へ**スケール**させる。「大規模組織では専門家個人が全てをリファクタリングするのは不可能」。専門家は**実行者でなく増幅装置**になる。

> 個々の設計技法は `mino-design-core` / `mino-domain-design` / `mino-code-design` / `mino-refactoring`。このスキルはそれらを「どう組織へ根付かせるか」を扱う。

## いつ・どう使うか

- 設計勉強会／輪読会を設計するとき → `design-learning-workshop`（ハンズオン8ステップ）
- チーム・組織の設計力を底上げする施策を組むとき → `design-knowledge-scaling`（設計責任者育成・レビュー文化・乳化）

## クイック原則（本人語彙）
- 「自転車の乗り方を本で読んでも乗れない」— ハンズオンが必須（読書だけは定着10%、言語化＋手を動かすと90%）
- 「技術は問題解決の手段」「理想構造を知らないとギャップに気づけない」
- 「わずか15時間（予習1h＋実習2h×5回）で実践レベル」
- 「120人超のエンジニア一人ひとりが設計スキルを持つ」＝スケールの発想
- 「負債解消とは"あるべき姿"を定義すること」
- 「乳化」＝エンジニアリングが組織に自然に溶け込む状態
- 「選択と集中」— 全面改善を一気にやらない（過去の失敗から得た教訓）
