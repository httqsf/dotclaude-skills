# t-wada-test-reliability — テストの信頼性（嘘をつかせない）

t-wada（和田卓人）の gihyo連載「**サバンナ便り**」の中心テーマ。自動テストの価値は**結果に嘘がないこと**——「成功→リリースOK／失敗→コードを直す」の二択が常に成り立つことにある。この二択を壊す4現象を潰す。

> 核: 全テスト実行時、**成功も失敗もどちらも信頼できる**状態を保つ。テストが信用できなくなると、組織全体が緑/赤を見なくなる。

## いつ・どう使うか（4現象の診断）

| 症状 | 現象 | 対処 |
|---|---|---|
| 通るのにバグがすり抜ける | 偽陰性（自作自演・空振り・skip放置） | `false-positive-negative` |
| コードを触っていないのに時々落ちる | 信頼不能（flaky） | `flaky-tests` |
| コードを少し変えると大量に壊れる | 脆いテスト（過結合） | `fragile-tests` |
| 実行順・並列で結果が変わる | 独立性・決定性の欠如 | `test-isolation-determinism` |

## クイック原則
- 偽陽性＝火が無いのに鳴る／偽陰性＝火が有るのに鳴らない（**偽陰性が静かで恐ろしい**）
- 期待値をプロダクトと同じロジックで計算しない（**自作自演**でバグが相殺される）
- flaky と判明したら「flakyタグ」で**メインCIから隔離**→修復→復帰
- **flaky＝触れてないのに揺れる／脆い＝触れると壊れる**（原因が違う）
- テスト失敗に**必ずアクションを取る**（狼少年化＝失敗への鈍感さを防ぐ）
