# test-isolation-determinism — 独立性・並列実行・決定性

## 核心
決定性とは「外部環境に結果が左右されず、同じ条件で常に同じ結果を返す」性質。t-wadaは連載第4回で、テストダブルを**「忠実性（本物らしさ）を下げる代わりに速度と決定性を上げる技術」**と定義し、これを「トレードオフであり正解はない」テスト設計判断として提示する。決定性の高いテストはメモリ上で外部リソースを使わずに動くため、**テスト間の依存を排除でき、実行順序に左右されず、並列実行が可能**になる——これが信頼性・高速フィードバックの土台。

## 指針
- 非決定要因（**ランダム・現在時刻・外部サービス通信・ネットワーク・DB・ファイル・並行処理**）をテストダブルや依存注入で制御可能にし、決定性を確保する。
- 決定性を上げる目的でテストダブルを使う。ただし**すでに決定性が高いテストを「速くするためだけ」にはテストダブルを使わない**（忠実性を無駄に犠牲にしない）。
- テストを**互いに独立**させる。各テストは他テストの副作用や実行順に依存してはならない（順序非依存）。
- **共有リソースを持ち込まない**。外部リソースを使わずメモリ上で完結させれば並列実行が容易になる（shared dependencyを断つ）。避けられない場合はテストごとに分離・後始末する。
- 忠実性と決定性のトレードオフを意識してテストサイズを選ぶ（→ `t-wada-test-strategy`）。

## アンチパターン
- 現在時刻・乱数・実ネットワーク/DBに直結した非決定テスト
- テスト間で状態・フィクスチャを共有して実行順序に依存する構造
- 相互干渉テスト（Interacting Tests）／リソースリークによる後続テスト汚染
